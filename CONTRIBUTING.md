# Contributing to AI Review Agent

First off, thank you for considering contributing to the AI Review Agent! It's people like you that make AI-powered development tools better.

## Where do I go from here?

If you've noticed a bug or have a feature request, please make sure it hasn't been reported already by checking the repository's open **Issues**.

If it's a new issue or feature, feel free to open a new one. Provide as much context as possible.

## Development Setup

1. Fork the repo and clone it locally.
2. Install dependencies: `go mod download`
3. Set up your `.env` following `.env.example`.
4. Ensure you can build both binaries smoothly:
   ```bash
   go build -o server ./cmd/server
   go build -o cli ./cmd/cli
   ```

## Coding Conventions

- **Language Details**: This project follows Go standard guidelines. Use `gofmt` or `goimports` to format your code before creating a MR.
- **Testing**: Whenever you add a feature or fix a bug, please add or update the corresponding tests. Currently, focus on unit-testing new logic placed under `internal/core` or `internal/pkg`.
- **Naming**: Use clear naming conventions in camelCase or PascalCase. Follow the Go standard library conventions for variable naming and scopes.

## Branching & Merge Requests

- Create a feature branch off of the `main` branch.
  Example: `feature/add-new-llm-provider` or `bugfix/fix-webhook-panic`.
- Write clear and descriptive commit messages.
- Open a Merge Request against the `main` branch of the original repository.
- Provide a clear PR description:
  - What issue does this fix?
  - What are the changes involved?
  - How can it be tested?

When your MR is reviewed, the maintainer may leave feedback (possibly generated by this very agent!). Address the comments and push.

We look forward to reviewing your pull requests!
